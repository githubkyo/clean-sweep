# 要件定義書

## プロジェクト概要
CleanSweep - macOS開発者向けストレージクリーナー。Docker/Xcode/npm/Gradle/AIツール等の一時ファイル・キャッシュを可視化し、安全度スコア付きで削除できるSwiftUIネイティブアプリ。APFSスナップショットや/private/tmpなどの隠れた巨大ファイルも検出する。

## 機能要件

### REQ-1: ストレージスキャン
**When** ユーザーが「スキャン」ボタンを押した時、**the system shall** 以下のカテゴリのディレクトリを走査し、各項目のパス・サイズ・カテゴリ・安全度を収集する:
- Docker（イメージ、ボリューム、ビルドキャッシュ）
- Xcode（DerivedData、Archives、iOS DeviceSupport、CoreSimulator）
- Node.js（npm/yarn/pnpmキャッシュ、node_modules、nvm）
- Flutter（pub-cache、CocoaPods）
- Gradle（.gradleキャッシュ）
- AIツール（Claude Code一時ファイル、Claude Desktopデータ）
- システムキャッシュ（Chrome、Homebrew、VS Code、pip）
- 一時ファイル（/private/tmp配下で100MB以上のエントリ）
- APFSスナップショット（diskutil経由で列挙）

**検証基準**: 9カテゴリ全てがスキャン対象に含まれ、各項目にパス・サイズ・カテゴリ・安全度が設定されること。

### REQ-2: 安全度分類
**The system shall** 各検出項目に以下の3段階の安全度レベルを割り当てる:
- **安全（Safe）**: 削除しても自動再生成されるもの（例: DerivedData、npmキャッシュ）
- **注意（Caution）**: 削除前に確認が推奨されるもの（例: Archives、iOS DeviceSupport）
- **危険（Dangerous）**: 削除するとデータ損失の可能性があるもの（例: Dockerディスクイメージ全体）

**検証基準**: 全項目に安全度が割り当てられ、UIで色分け表示されること。

### REQ-3: カテゴリ別表示
**The system shall** スキャン結果をカテゴリ別にグループ化し、各カテゴリの合計サイズとアイテム数を表示する。カテゴリはサイズの大きい順にソートされる。

**検証基準**: カテゴリがサイズ降順で表示され、各カテゴリに合計サイズとアイテム数が表示されること。

### REQ-4: 選択と削除
**When** ユーザーが項目を選択して「削除」ボタンを押した時、**the system shall** 選択された項目をファイルシステムから削除し、削除結果（成功/失敗）をログとして表示する。

**検証基準**: 選択項目が削除され、成功・失敗がログに記録されること。削除失敗時にエラーメッセージが表示されること。

### REQ-5: 自動選択
**When** スキャンが完了した時、**the system shall** 安全度が「安全（Safe）」の項目を自動的に選択状態にする。「注意」「危険」の項目は未選択のまま残す。

**検証基準**: スキャン完了後、Safe項目のみが選択状態であること。

### REQ-6: カテゴリ一括選択/解除
**When** ユーザーがカテゴリヘッダーの選択チェックボックスを操作した時、**the system shall** そのカテゴリ内の全項目の選択状態を一括で切り替える。

**検証基準**: カテゴリチェックボックス操作で配下の全項目が連動すること。

### REQ-7: サイズフォーマット
**The system shall** 全てのファイルサイズをByte単位からヒューマンリーダブル形式（KB/MB/GB/TB）に自動変換して表示する。

**検証基準**: 全サイズ表示が適切な単位で表示されること（例: 1073741824B → "1.07 GB"）。

### REQ-8: スキャン進捗表示
**While** スキャンが実行中の間、**the system shall** 現在スキャン中のカテゴリ名をプログレス表示として画面に表示する。

**検証基準**: スキャン中にカテゴリ名が逐次更新表示されること。

### REQ-9: 削除確認ダイアログ
**When** ユーザーが削除ボタンを押した時、**the system shall** 選択項目数と合計サイズを含む確認ダイアログを表示し、ユーザーが明示的に承認した場合のみ削除を実行する。

**検証基準**: 確認ダイアログなしに削除が実行されないこと。

### REQ-10: 合計回収可能サイズ表示
**The system shall** スキャン結果の合計回収可能サイズと、現在選択中の項目の合計サイズをヘッダー領域に常時表示する。

**検証基準**: 合計サイズと選択サイズが常に表示され、選択変更時にリアルタイム更新されること。

## 非機能要件

### NFR-1: パフォーマンス
**The system shall** ホームディレクトリ配下のスキャンを30秒以内に完了する（SSD環境、ファイル数100万未満）。

### NFR-2: プラットフォーム
**The system shall** macOS 14 (Sonoma) 以上で動作するSwiftUIネイティブアプリケーションとして実装される。Swift Package Managerでビルド可能であること。

### NFR-3: 安全性
**The system shall** ファイル削除前に必ず確認ダイアログを表示し、危険レベルの項目は赤色で強調表示する。sudo権限が必要な操作（APFSスナップショット削除）は、必要なコマンドをクリップボードにコピーする機能で対応する。

### NFR-4: メモリ使用量
**The system shall** スキャン中のメモリ使用量を200MB以下に抑える。ディレクトリサイズ計算は`du`コマンド経由で行い、再帰的なファイル列挙を避ける。

### NFR-5: エラー耐性
**When** スキャン中にアクセス権限エラーが発生した場合、**the system shall** そのディレクトリをスキップし、スキャンを続行する。エラーが発生したディレクトリはログに記録する。
